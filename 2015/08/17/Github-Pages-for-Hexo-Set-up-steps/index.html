GitHub Pages和Hexo搭建无限流量之静态独立Blog

发表于 2015-08-17   |   分类于 Conding   |  

前言

作为一名技术控,有必要搭建一个属于自己的独立博客,用来记录学习过程中的点点滴滴,同世界一起见证自己的成长 .

为什么选择Github Pages搭建独立博客呢 ???

  1. Github号称”全球最大的同志交友网站”,截至2015年,拥有900万注册用户和2110 万代码仓库,事实上已经成为全球最大的代码存放网站,其中不乏大牛级人物 .
  2. Github完美支持Markdown标记语法.让我们更专注于写字本身而非排版样式 .
  3. 拥有300M空间和无限流量的Github. Page足以满足个人博客的需求 .独立的东西才是属于自己的 .
  4. 顺便了解Github的工作原理,具备最好的团队协作流程 .
  5. 如果Github不幸被墙,分分钟转移国内Gitcafe .
  6. 支持独立域名的绑定 .
  7. Github是一种趋势 .
    以上链接需科学上网

准备工作

一次下载安装以下软件

  1. Node.js
  2. Git

注意: Nodes.js默认安装完成后需添加Path环境变量，使npm命令生效

;C:\Program Files\nodejs\node_modules\npm;

如果Node.js非默认安装路径(D:\Code\Node),则变量值如下

;C:\Program Files\nodejs\node_modules\npm;D:\Code\Node

如何打开Git???

  1. 开始菜单-所有程序-Git

注册GitHub

访问:https://github.com/

注册过程比较简单,无需多言

username和邮箱，其中邮箱十分重要，GitHub上很多通知都是通过邮箱

为什么要配置SSH Keys ???

让本地Git项目与远程的github建立联系

检查SSH keys的设置

首先我们需要检查你电脑上现有的ssh key：

$ cd ~/. ssh 检查本机的ssh密钥

如果提示：No such file or directory 说明你是第一次使用git .

生成新的SSH Key：

$ ssh-keygen -t rsa -C “邮件地址@youremail.com”
Generating public/private rsa key pair .
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa): <回车就好>

注意1: 此处的邮箱地址，你可以输入自己的邮箱地址；注意2: 此处的 -C 的是大写的 C

然后系统会要你输入密码：

Enter passphrase (empty for no passphrase):<输入密码>
Enter same passphrase again:<再次输入密码>

在提交项目时需使用该密码，如果为空则提交项目时不需要输入 .这样做的好处是防止别人往你的项目里提交内容

注意：输入密码的时候没有*字样的，直接输入就行.

最后看到这样的界面，就成功设置ssh key了：

添加SSH Key到GitHub

本机设置SSH Key之后，需要添加到GitHub上，以完成SSH链接的设置

  1. 打开本地C:\Users\Administrator\.ssh\id_rsa.pub文件. 此文件内容便是密钥. 如果看不到这个文件，则需要设置显示隐藏文件以及显示已知文件的扩展名 .然后用Sublime打开该文件,然后复制里面的内容.
  2. 登陆github. 点击右上角的 Settings—> SSH Keys —> Add SSH Key .
  3. 将密钥复制到里面（key文本框中）， 点击 Add key 就ok了 .

测试

输入下面的命令，查看设置是否成功:

$ ssh -T git@github.com

如果是下面的反馈：

Enter passphrase for key ‘/c/Users/Divergent/.ssh/id_rsa’: (键入密码)

没关系，输入yes好了，然后会看到：

Hi RuleGod! You’ve successfully authenticated, but GitHub does not provide shell access.

设置用户信息

现在可以通过SSH连接到GitHub，然而还需要完善一些个人信息.

Git会根据用户的名字和邮箱来记录提交 . GitHub也是用这些信息来判断权限大小，输入以下代码进行个人信息的设置,名称和邮箱替换成你自己的.

$ git config –global user.name “username”

$ git config –global user.email “user_name@yourname.com”

SSH Key配置成功

本机已成功连接到github .

若有问题，请重新设置.常见错误请参考：

GitHub Help - Generating SSH Keys

GitHub Help - Error Permission denied (publickey)

利用GitHub Pages搭建博客

与GitHub建立好链接之后，就可以方便的使用它提供的Pages服务，GitHub Pages分两种，一种是你的GitHub用户名建立的username.github.io这样的用户&组织页（站），另一种是依附项目的pages .

想建立个人博客是用的第一种，形如RuleGod.github.io这样的可访问的站，每个用户名下面只能建立一个 .

github上建立仓库

登录GitHub系统，在github首页，点击页面右下角New Repository

填写项目信息：

Repository name：username.github.io

注：Github Pages的Repository名字是特定的
示例:Github账号是RuleGod，则Github Pages Repository名字就是RuleGod.github.io

点击 Create Repository 完成创建

用Hexo克隆主题

Hexo介绍

Hexo的作者是tommy351，据官方介绍，Hexo是一款简单、快速、强大的博客发布工具，支持Markdown语法.

安装Hexo

打开 git ,键入以下代码

npm install -g hexo

部署Hexo

在电脑上建立一个名叫 Hexo 的文件夹，然后在此文件夹上右键打开Git Bash ,键入以下代码

$ hexo init

Hexo随后会自动在目标文件夹建立网站所需要的所有文件 .

现在我们已经搭建起本地的hexo博客了，执行以下命令(在H:\hexo)，然后到浏览器输入localhost:4000看看 .

$ hexo g #生成

$ hexo s #启动本地服务，进行文章预览调试

部署到Github前准备工作

修改站点目录下的_config

deploy:
type: git
repository: git@github.com:AlanCheen/alancheen.github.io.git /SSH链接
branch: master
message:

新教程中type: git

安装git插件 hexo-deployer-git

$ npm install hexo-deployer-git –save

开始部署

$ hexo d

如果一切顺利,你就能看到自己的博客

复制jacman主题

博客根目录下执行如下命令

$ git clone https://github.com/wuchong/jacman.git themes/jacman

注: Jacman 需要安装 Hexo 2.7 及以上版本，请先升级 Hexo 程序，再启用此主题

启用主题

修改站点目录下的_config.yml配置文件中的 theme 属性，将将其修改为jacman . 同时建议修改stylus属性中的compress值为true，自动压缩 CSS 文件，hexo 默认配置中不包含这一项，建议开启 .

示例:

theme: jacman
stylus:
compress: true

更新

cd themes/jacman
git pull origin master

注:请先备份主题目录下的 _config.yml 文件后再升级

本地查看调试

$ hexo g #生成

$ hexo s #启动本地服务，进行文章预览调试

浏览器访问http://localhost:4000，查看搭建效果.此后每次变更_config.yml文件或者上传文件都可以先用此命令调试，非常好用，尤其是当你想调试出自己想要的主题 .

将独立域名与GitHub Pages的空间绑定

GitHub Pages的设置

方法一：在Repository的根目录下面，新建一个名为CNAME的文本文件，里面写入你要绑定的域名，比如 rulegod.me .

方法二：在本地文件Hexo\source目录下新建 CNAME 文件，用编辑器打开此文件, 填写你要访问的域名,然后用hexo命令提交便可 .

示例:

提交命令如下:

$ hexo d -g

域名绑定

登录Godaddy,点击 My Account —> 用户名

点击 DOMAINS , 选择要绑定的域名, 然后点击 Mange

选择 DNS ZONE FILE 点击 Add Record 添加 A记录

其中A的两条记录指向的ip地址是github Pages的提供的ip

  • 192.30.252.153
  • 192.30.252.154

注意:博客不能登录，有可能是github更改了空间服务的ip地址，记得及时到在GitHub Pages查看最新的ip即可
